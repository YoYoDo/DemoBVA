## 🏆 核心特性

### 1. 三層自動化架構

**第一層**: 環境配置 (.env)
- 30+ 個環境變數
- 一次配置，全局生效
- 支持手動和自動模式切換

**第二層**: 初始化引擎 (startup.js)
- 自動掃描技能和工作流
- 智能恢復工作狀態
- 生成可視化狀態報告
- 398 行精優化代碼

**第三層**: 三大系統集成
- 技能系統（發現和管理）
- 工作流系統（定義和執行）
- 記憶系統（學習和積累）

### 2. 智能記憶系統

8 個專用記憶檔案，各司其職：

```
learned-skills.md         ← 技能索引（掃描時更新）
workflow-history.md       ← 工作流執行歷史（完成時更新）
active-workflow-state.md  ← 當前工作流狀態（實時更新）
session-history.md        ← 任務完成記錄（任務完成時更新）
context-cache.md          ← 工作上下文（狀態改變時更新）
task-status.md            ← 任務追蹤（任務改變時更新）
discoveries.md            ← 發現和最佳實踐（發現時更新）
QUICKREF.md               ← 快速參考卡（手動維護）
```

### 3. 完整工作流系統

支持建立、追蹤、恢復和優化工作流：

```
工作流定義
  ↓ WORKFLOW.md         (工作流說明和步驟)
  ↓ steps.json          (結構化步驟配置)
  ↓ checklist.md        (完成檢查清單)
  ↓ templates/          (可複用模板)
  
工作流執行
  ├─ 進度追蹤           (active-workflow-state.md)
  ├─ 中斷恢復           (保存執行上下文)
  └─ 完成記錄           (workflow-history.md)
```

### 4. 開箱即用的提示詞

5 個完整的提示詞模板，複製即用：

```
[系統初始化]           ← 首次啟動
[執行工作流: xxx]      ← 執行工作流
[恢復工作流: xxx]      ← 恢復中斷
[任務完成]             ← 更新記憶
[查詢記憶]             ← 查詢和應用
```

每個提示詞都包含詳細說明和格式要求。

## 🚀 立即開始（3 步驟）

### 第 1 步: 執行系統初始化

**方式 A - 在 Claude Code 中使用提示詞** (推薦)

複製以下文本到 Claude Code：

```
[系統初始化]

請立即執行以下步驟：

1. **掃描並學習所有技能**
   - 列出 .claude/skills/ 中的所有技能
   - 為每個技能解析關鍵信息
   - 已學習的技能更新到 learned-skills.md

2. **掃描並註冊所有工作流**
   - 列出 .claude/workflows/ 中的所有工作流
   - 為每個工作流解析定義
   - 將工作流註冊到 active-workflow.md

3. **恢復工作上下文**
   - 讀取 context-cache.md
   - 讀取 active-workflow-state.md（檢查有無中斷的工作流）
   - 讀取 task-status.md（顯示所有未完成任務）
   - 讀取 session-history.md（顯示最近 3 項任務）

4. **準備工作**
   - 讀取 discoveries.md（記住之前的解決方案）
   - 執行完初始化後，輸出一份精簡的狀態報告

格式要求：
- 技能列表：使用表格或列表
- 工作流列表：突出可執行的工作流
- 待辦任務：突出 high priority 任務
- 活躍工作流：如有則顯示進度
- 狀態報告：簡明扼要，不超過 15 行

初始化完成後，我會提出具體需求。
```

## 🔧 故障排除

### 初始化失敗

1. 確認 .env 檔案存在
2. 檢查 Node.js 版本：`node -v`
3. 查看詳細日誌：設置 CLAUDE_VERBOSE_INIT=true
4. 查看 PROMPTS.md 的故障排除部分

### 工作流執行問題

1. 檢查工作流定義是否有效
2. 驗證所有先決條件已滿足
3. 查看 active-workflow-state.md 中的當前狀態
4. 參考 workflows/README.md 進行調試

### 記憶檔案問題

1. 確認記憶檔案存在於 .claude/memory/
2. 檢查檔案權限
3. 確保 CLAUDE_MEMORY_ENABLED=true
4. 查看 config/init.md 的記憶系統說明
