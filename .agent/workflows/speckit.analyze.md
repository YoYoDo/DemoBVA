# /speckit.analyze - 規格一致性分析

## 用途

識別三個核心產出物（spec.md、plan.md、plan-uiux.md、tasks.md）之間的不一致、重複、歧義和未充分指定的項目。此命令**必須**在 `/speckit.tasks` 成功產生完整 tasks.md 後才能執行。

## 操作約束

**嚴格唯讀**: **不**修改任何檔案。輸出結構化分析報告。提供可選的修復計畫（使用者必須明確批准後才能手動調用任何後續編輯命令）。

**憲章權威**: 專案憲章 (`.specify/memory/constitution.md`) 在此分析範圍內**不可協商**。憲章衝突自動為 CRITICAL，需要調整規格、計畫或任務。

## 執行步驟

### 1. 初始化分析上下文

```bash
# turbo
.specify/scripts/bash/check-prerequisites.sh --json --require-tasks --include-tasks
```

解析 FEATURE_DIR 和 AVAILABLE_DOCS。推導絕對路徑：

- SPEC = FEATURE_DIR/spec.md
- PLAN = FEATURE_DIR/plan.md
- PLANUIUX = FEATURE_DIR/plan-uiux.md
- TASKS = FEATURE_DIR/tasks.md

如果任何必需檔案缺失則中止。

### 2. 載入產出物（漸進式揭示）

**從 spec.md**:

- 概述/上下文、功能需求、非功能需求、使用者故事、邊緣案例

**從 plan.md**:

- 架構/堆疊選擇、資料模型引用、階段、技術約束

**從 plan-uiux.md**:

- 畫面欄位確認、資料類別檢查、儲存檢查、使用體驗優化

**從 tasks.md**:

- 任務 ID、描述、階段分組、平行標記 [P]、引用的檔案路徑

**從憲章**:

- 載入 `.specify/memory/constitution.md` 進行原則驗證

### 3. 建構語義模型

建立內部表示（不在輸出中包含原始產出物）：

- **需求清單**: 每個功能 + 非功能需求
- **使用者故事/動作清單**: 離散使用者動作與驗收標準
- **任務覆蓋對應**: 對應每個任務到一個或多個需求/故事
- **憲章規則集**: 提取原則名稱和 MUST/SHOULD 規範陳述

### 4. 偵測通道（Token 高效分析）

專注於高訊號發現。限制總計 50 個發現。

#### A. 重複偵測

- 識別近重複需求
- 標記較低品質措辭以進行整合

#### B. 歧義偵測

- 標記缺乏可衡量標準的模糊形容詞（快速、可擴展、安全、直覺、健壯）
- 標記未解決的佔位符（TODO, TKTK, ???）

#### C. 未充分指定

- 有動詞但缺少物件或可衡量結果的需求
- 缺少驗收標準對齊的使用者故事
- 引用未在 spec/plan 中定義的檔案或元件的任務

#### D. 憲章對齊

- 任何與 MUST 原則衝突的需求或計畫元素
- 憲章中缺少的強制章節或品質閘門

#### E. 覆蓋差距

- 零相關任務的需求
- 無對應需求/故事的任務
- 未反映在任務中的非功能需求

#### F. 不一致

- 術語漂移（相同概念在不同檔案中有不同名稱）
- 計畫中引用但規格中缺少的資料實體（反之亦然）
- 任務排序矛盾

### 5. 嚴重性分配

- **CRITICAL**: 違反憲章 MUST、缺少核心規格產出物、阻塞基線功能的零覆蓋需求
- **HIGH**: 重複或衝突需求、歧義安全/效能屬性、不可測試驗收標準
- **MEDIUM**: 術語漂移、缺少非功能任務覆蓋、未充分指定的邊緣案例
- **LOW**: 風格/措辭改進、不影響執行順序的輕微冗餘

### 6. 產生精簡分析報告

輸出 Markdown 報告（不寫入檔案）：

| ID | Category | Severity | Location(s) | Summary | Recommendation |
|----|----------|----------|-------------|---------|----------------|

**覆蓋摘要表**、**憲章對齊問題**、**未對應任務**、**指標**

### 7. 提供下一步行動

- 如果存在 CRITICAL 問題：建議在 `/speckit.implement` 之前解決
- 如果只有 LOW/MEDIUM：使用者可繼續，但提供改進建議

### 8. 提供修復建議

詢問使用者：「您希望我為前 N 個問題建議具體的修復編輯嗎？」（不自動應用）
