# Plan 1 - 投資撥回試算與比較系統

## 目標
依據 `spec/specify.md`，完成可上線的單頁試算工具，涵蓋雙情境撥回計算、匯率即時連動、匯出/匯入、列印與可用性優化。

## 範圍
- 前端單頁應用（SPA）
- 核心計算與資料彙總
- 表格與儀表板呈現
- 匯入/匯出與列印
- Demo 資料與驗收測試

## 里程碑

### M1. 核心計算模型
- 建立/確認 `calculatePayout` 雙情境輸出：
  - 情境 A：NAV >= 9，因子 0.042
  - 情境 B：8 <= NAV < 9，因子 0.036
- 完成以下欄位計算：
  - 實際單位數
  - 月撥回（TWD）
  - 年撥回（TWD）
  - 累積平均單位成本
- 顯示層格式化：千分位與小數顯示切換（整數 / 小數 2 位）

### M2. 輸入流程與狀態管理
- 快速新增表單：保費、費用率、進場匯率、NAV
- 現時匯率設為全域參數並即時連動所有列結果
- 每列支援編輯與刪除
- 提供「清除全部」
- 預設載入策略：頁面重整後不保留舊資料（以規格為準）

### M3. 儀表板與表格
- 組合總覽卡片：
  - 總投入本金
  - 加權平均單位成本
  - 情境 A / B 的月撥回與年撥回總額
- 動態表格欄位：
  - 實際單位數
  - 情境 A 月/年撥回
  - 情境 B 月/年撥回
  - 平均單位成本
- NAV 視覺標示：
  - NAV < 8 藍色
  - 8 <= NAV < 9 黃色
  - NAV >= 9 綠色

### M4. 匯出、匯入與列印
- Excel 匯出（.xlsx，SheetJS）
- Excel 匯入（可完整還原資料列）
- 全頁列印
- 欄位 Tooltip 說明（含單位與公式）

### M5. 品質與驗收
- 單元測試：`calculatePayout`（含雙情境與邊界值）
- 驗證 Demo 數據與規格一致
- RWD（桌機/平板/手機）
- Keyboard navigation 與 ARIA（WCAG AA 基本要求）

## 風險與處理
- 規格衝突：`specify.md` 同時提到「不保留資料」與 `localStorage`。
  - 處理：以「不保留資料」為主，若需保留改為可切換設定。
- 匯入資料欄位版本差異。
  - 處理：匯入時加欄位映射與必要欄位驗證。
- 計算精度與顯示精度不一致。
  - 處理：內部維持浮點精度，僅在 UI 格式化時四捨五入。
